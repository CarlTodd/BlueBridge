# P3958 [NOIP 2017 提高组] 奶酪

## 题目背景

NOIP2017 提高组 D2T1

## 题目描述

现有一块大奶酪，它的高度为 $h$，它的长度和宽度我们可以认为是无限大的，奶酪中间有许多半径相同的球形空洞。我们可以在这块奶酪中建立空间坐标系，在坐标系中，奶酪的下表面为 $z = 0$，奶酪的上表面为 $z = h$。

现在，奶酪的下表面有一只小老鼠 Jerry，它知道奶酪中所有空洞的球心所在的坐标。如果两个空洞相切或是相交，则 Jerry 可以从其中一个空洞跑到另一个空洞，特别地，如果一个空洞与下表面相切或是相交，Jerry 则可以从奶酪下表面跑进空洞；如果一个空洞与上表面相切或是相交，Jerry 则可以从空洞跑到奶酪上表面。

位于奶酪下表面的 Jerry 想知道，在不破坏奶酪的情况下，能否利用已有的空洞跑到奶酪的上表面去?

空间内两点 $P_1(x_1,y_1,z_1)$、$P_2(x_2,y_2,z_2)$ 的距离公式如下：


$$\mathrm{dist}(P_1,P_2)=\sqrt{(x_1-x_2)^2+(y_1-y_2)^2+(z_1-z_2)^2}$$

## 输入格式

每个输入文件包含多组数据。

第一行，包含一个正整数 $T$，代表该输入文件中所含的数据组数。

接下来是 $T$ 组数据，每组数据的格式如下： 第一行包含三个正整数 $n,h,r$，两个数之间以一个空格分开，分别代表奶酪中空洞的数量，奶酪的高度和空洞的半径。

接下来的 $n$ 行，每行包含三个整数 $x,y,z$，两个数之间以一个空格分开，表示空洞球心坐标为 $(x,y,z)$。



### 2. 算法选择
正如你所想，有两种非常高效的方法：

*   **BFS/DFS（广度优先/深度优先搜索）**：
    *   **起点**：先遍历一遍所有球体，将满足 $z_i \le r$（即与下表面相切或相交）的球加入搜索队列或栈中。
    *   **过程**：从起点开始扩散，如果当前球与下一个球的距离 $\le 2r$，则继续搜索。
    *   **终点**：在搜索过程中，如果碰到满足 $z_i + r \ge h$ 的球，直接返回 `Yes`。
*   **并查集 (Union-Find)**（非常推荐）：
    *   这种方法对于“连通性”判断非常优雅。
    *   你可以创建两个虚拟节点：一个代表“下表面”，一个代表“上表面”。
    *   将所有与下表面相连的球并入“下表面”集合，所有与上表面相连的球并入“上表面”集合。
    *   如果两个球相交，则把它们所在集合合并。
    *   最后只需判断“下表面”集合和“上表面”集合是否合并为了同一个集合（或者检查是否有球同时满足连接上下表面）。


### 2. 并查集逻辑（静态焊接）：
并查集像是一个**工程师**。他手里拿着一个图纸，上面有所有洞的坐标。他做的事情是：
*   **第一步（焊接）**：遍历所有洞，把所有满足“距离 $\le 2r$”的洞两两焊接起来。
*   **第二步（检查）**：焊接完成后，他看一眼图纸，问：“现在下表面和上表面是不是被焊在同一个大铁块上了？”

**为什么“遍历一遍”就够了？**

因为“连通”是一个**结果性**的属性，而不是一个“过程”属性。

想象一下，如果有 3 个洞 A、B、C，路径是：下表面 $\to$ C $\to$ B $\to$ A $\to$ 上表面。

当你执行“一重遍历”（其实是双重循环遍历所有两两组合）时，程序会发生这样的事：
1.  程序发现 C 和下表面相连，焊接：`Union(C, 下表面)`
2.  程序发现 C 和 B 相连，焊接：`Union(C, B)`
3.  程序发现 B 和 A 相连，焊接：`Union(B, A)`
4.  程序发现 A 和上表面相连，焊接：`Union(A, 上表面)`

**关键点在于：** 无论程序先处理哪一对（是先处理 B-A，还是先处理 C-B），**并查集都会把它们记录在案**。由于并查集有“路径压缩”和“合并”功能，不管你遍历的顺序如何，只要这些“焊接点”都在你的遍历范围内，最终这几个洞一定会合并成一个集合。

### 为什么不需要“重新遍历”或“不断搜索”？
因为我们并不关心路径长什么样，我们只关心**“它们是否属于同一个整体”**。

*   如果我们遍历了所有的组合（即 $N \times N$ 的配对），这意味着**所有潜在的连接机会我们都检查过了**。
*   既然所有机会都检查过了，如果还存在一条从下到上的路径，那么所有的点一定已经被焊接进了同一个集合。
*   如果遍历完所有组合，下表面和上表面还是不连通（`Find(下) != Find(上)`），那说明无论你怎么走，都不可能走通。

**没错，这就是最标准、最高效的做法！**

### 代码实现的核心思路
你只需要用两层循环把所有“可能相连的球”都检查一遍，并放入并查集（Union-Find）中。代码的结构大致如下：

```cpp
// 1. 初始化并查集
for(int i = 0; i < n; i++) parent[i] = i;

// 2. 遍历所有球，处理与上下表面的关系
for(int i = 0; i < n; i++) {
    if (z[i] <= r) unite(i, BOTTOM);      // 与下表面连接
    if (z[i] + r >= h) unite(i, TOP);    // 与上表面连接
}

// 3. 两重循环，遍历所有球对，处理球与球的关系
for(int i = 0; i < n; i++) {
    for(int j = i + 1; j < n; j++) {     // j 从 i+1 开始，避免重复判断(i,j)和(j,i)
        if (dist_sq(i, j) <= 4 * r * r) {
            unite(i, j);
        }
    }
}

// 4. 最后判断
if (find(BOTTOM) == find(TOP)) cout << "Yes";
else cout << "No";
```
